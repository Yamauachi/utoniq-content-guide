# utoniq コンテンツガイド

utoniq の UI デザインにサンプルテキストを挿入するためのガイドです。
AI（Claude 等）と組み合わせて使うことで、誰でも統一感のあるサンプルテキストを作成できます。

---

## ファイル一覧

| ファイル | 内容 |
|---|---|
| [persona.md](persona.md) | ペルソナ設定書 — アーティスト・ファンの人物設定、話し方、数値基準 |
| [tone-table.md](tone-table.md) | テキストトーン表 — 機能カテゴリ別のテキスト方針、文字数目安 |
| [profile-images.md](profile-images.md) | プロフィール画像URLマッピング — 各キャラクターの固定画像URL |
| [artists-list.md](artists-list.md) | サンプルアーティスト名リスト — 一覧画面等で使う追加アーティスト10組 |

---

## AI を使ったサンプルテキスト作成フロー

### Step 1：AI にファイルを読み込ませる

Claude などの AI チャットで、以下のように指示します。

```
以下のドキュメントを読み込んでください。
これはアプリのUIに挿入するサンプルテキストの基準です。

[persona.md の内容を貼り付け]
[tone-table.md の内容を貼り付け]
[profile-images.md の内容を貼り付け]
[artists-list.md の内容を貼り付け]
```

> **Tips**：Claude Code や ChatGPT であればファイルを直接添付できます。GitHub の Raw URL を渡す方法もあります。

### Step 2：対象画面のスクリーンショットを共有する

テキストを入れたい Figma 画面のスクリーンショットを AI に送ります。

```
この画面にサンプルテキストを入れたいです。
ペルソナとトーン表に従って、各テキスト要素の内容を提案してください。
```

### Step 3：AI が画面の種類を判定し、テキストを生成する

AI は以下を自動的に行います：

1. **画面の機能カテゴリを特定**（例：ポスト、ガチャ、イベント…）
2. **トーン表から該当カテゴリのルールを参照**（話者、トーン、文字数）
3. **ペルソナに基づいてテキストを生成**（キャラの話し方、数値の基準値）

### Step 4：テキストを確認して Figma に適用する

AI の提案を確認し、必要に応じて調整してから Figma に反映します。

---

## 確認チェックリスト

テキスト適用後、以下を確認してください：

- [ ] アーティスト名は **Lumière**（ルミエール）を使っているか？
- [ ] メンバーの話し方がペルソナ通りか？（ハルト＝穏やか、レン＝頼れる兄貴、等）
- [ ] ファンのコメントは 3 人のペルソナ（夏希・りな・ケンタ）に基づいているか？
- [ ] 推し状態のデフォルトは「夏希 → ハルト」になっているか？
- [ ] 数値（フォロワー数、いいね数、コイン等）がペルソナの基準値に沿っているか？
- [ ] 新しいキャラクター名を勝手に作っていないか？
- [ ] テキストの文字数がトーン表の目安以内か？

---

## 注意事項

- **メインアーティストは常に Lumière を使用**してください。サブアーティスト（月島遥、藤宮朱音、THE ORBIT）はランキングや一覧画面の脇役として使います
- 追加アーティストが必要な場合は `artists-list.md` のリストから選んでください
- プロフィール画像が必要な場合は `profile-images.md` のURLを使用してください
- ペルソナに記載のない人物名・グループ名を新たに作らないでください
- 数値の「盛りすぎ」「少なすぎ」に注意。ペルソナの基準値を参照してください
- 不明点があればこのリポジトリのファイルを正として参照してください
